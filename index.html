A Flexible, Easy-To-Use DatePicker

Chronos was created to fill the need for an easy-to-use datepicker without having to rely on a picker such as jQueryUI and the associated overhead. Chronos is designed to be jQuery and Zepto compatible and follows the standard jQuery plugin pattern for interaction. 

$('#basic').chronos();

Under The Hood

When a chronos picker is initialized, the original input element (the 'value element') becomes hidden from the user, and a new element (the 'display' element) is created. The display element is a clone of the value element, but without the name attribute so that its value is not posted to the server. It is used for display purposes only. The display element's id consists of the id of the value element and "_display". In addition, the class 'chronos_picker' will be appended to the value element's classes. The class 'chronos_picker_display' will be appended to the display element.

Two input elements are used so that the display format of the chosen date is separated from the value posted to the server. When a date is selected, the value element is updated to the chosen date according to the format specified by the valueFormat option. The display element's value is updated according to the format specified by the displayFormat option.

Options

Chronos pickers may be adjusted on initialization by passing an options object.

Option Type Default Value Description
pickerClass String blank Optional string of classes to add to the display element.
dayNames String Array [ "Sun", "Mon", "Tue", "Wed", "Thu", "Fri", "Sat", "Sunday", "Monday", "Tuesday", "Wednesday", "Thursday", "Friday", "Saturday" ] Days of the week.
dayNamesAbbr String Array [ "Su", "Mo", "Tu", "We", "Th", "Fr", "Sa"] Abbreviated days of the week.
monthNames String Array [ "Jan", "Feb", "Mar", "Apr", "May", "Jun", "Jul", "Aug", "Sep", "Oct", "Nov", "Dec", "January", "February", "March", "April", "May", "June", "July", "August", "September", "October", "November", "December" ] Months of the year.
amLower String "am" Lowercase ante meridiem abbreviation.
amUpper String "AM" Uppercase ante meridiem abbreviation.
amAbbrLower String "a" Single character, lowercase ante meridiem abbreviation.
amAbbrUpper String "A" Single character, uppercase ante meridiem abbreviation.
pmLower String "pm" Lowercase post meridiem abbreviation.
pmUpper String "PM" Uppercase post meridiem abbreviation.
pmAbbrLower String "p" Single character, lowercase post meridiem abbreviation.
pmAbbrUpper String "P" Single character, uppercase post meridiem abbreviation.
startBlank Boolean false When true, the datepicker will start with a blank value, otherwise it will start with the current date or pickedDateTime.
displayFormat String "isoDate" Format to display after making a selection in the picker.
valueFormat String "U" Format of value posted to server.
maxDate Date undefined Maximum allowed date (inclusive). Must be a valid Date object.
minDate Date undefined Minimum allowed date (inclusive). Must be a valid Date object.
startDay Integer 0 Sunday (0) through Saturday (6) - be aware that this may affect your layout, since the days on the right might have a different margin.
pickedDateTime Date undefined Start datepicker at a specific date. Must be a valid Date object.
animations Object {previousMonth: undefined, nextMonth: undefined, close: undefined, open: undefined} Animations to use during the datepicker's operation. Object must consist of the given keys where the values are valid Functions. Default animations are performed when not defined by the user.
positionOffset Object {top: 0, left: 0} Offset to adjust position of picker. Units should be given as integers representing pixels (i.e. do not provide the unit 'px')
debug Boolean false Display element is not hidden when set to 'true'.

Note that dayNames and monthNames must contain abbreviated and full representations. In addition, order in the dayNames, dayNamesAbbr, monthNames arrays is important for date formatting. Use the startDay option to offset the week's starting day for display in calendar.

Date Formats

Date formatting in Chronos is based on Steve Levithan's awesome Javascript Date Format library. Date objects are formatted according to a user specified mask. For example, "m/dd/yy" would result in "6/09/12". One of several named masks may also be used. For example, "isoDateTime" would result in "20012-06-09T17:33:21". These formats can be used for the displayFormat and valueFormat options.

Mask Description
d Day of the month as digits; no leading zero for single-digit days.
dd Day of the month as digits; leading zero for single-digit days.
ddd Day of the week as a three-letter abbreviation.
dddd Day of the week as its full name.
m Month as digits; no leading zero for single-digit months.
mm Month as digits; leading zero for single-digit months.
mmm Month as a three-letter abbreviation.
mmmm Month as its full name.
yy Year as last two digits; leading zero for years less than 10.
yyyy Year represented by four digits.
h Hours; no leading zero for single-digit hours (12-hour clock).
hh Hours; leading zero for single-digit hours (12-hour clock).
H Hours; no leading zero for single-digit hours (24-hour clock).
HH Hours; leading zero for single-digit hours (24-hour clock).
M Minutes; no leading zero for single-digit minutes.
MM Minutes; leading zero for single-digit minutes.
s Seconds; no leading zero for single-digit seconds.
ss Seconds; leading zero for single-digit seconds.
U Unix Time: the number of milliseconds since midnight of January 1, 1970
l or L Milliseconds. l gives 3 digits. L gives 2 digits.
t Lowercase, single-character time marker string: a or p.
tt Lowercase, two-character time marker string: am or pm.
T Uppercase, single-character time marker string: A or P.
TT Uppercase, two-character time marker string: AM or PM.
Z US timezone abbreviation, e.g. EST or MDT. With non-US timezones or in the Opera browser, the GMT/UTC offset is returned, e.g. GMT-0500
o GMT/UTC timezone offset, e.g. -0500 or +0230.
S The date's ordinal suffix (st, nd, rd, or th). Works well with d.
'…' or "…" Literal character sequence. Surrounding quotes are removed.
UTC: Must be the first four characters of the mask. Converts the date from local time to UTC/GMT/Zulu time before applying the mask. The "UTC:" prefix is removed.

Accessing Methods

Methods on Chronos can be accessed directly through the object or through the jQuery/Zepto selectors. Notice in the following example that the date ranges have been set through the setDateRange method and not during plugin initialization.

var direct = $("#w_methods").chronos(),
mind = new Date(),
maxd = new Date();
maxd.setDate(maxd.getDate()+7);
mind.setDate(mind.getDate()-7);
direct.setDateRange({minDate: mind}); // direct access
$("#w_methods").chronos("setDateRange", {maxDate: maxd}); // indirect access
Method Returns Description
setDateRange({range options}) this Set the maximum and/or minimum date range. Accepts a range object in the format: {minDate: Date, maxDate: Date}

Events

Chronos has a variety of events that are triggered throughout operation.

Event Data Description
daySelected (Event, Date Selected, Element Selected) When a calendar day is selected.
opened (Event) When the datepicker has finished the open animation.
closed (Event) When the datepicker has finished the close animation.
previousMonthFinished (Event) When the datepicker has finished the previous month animation.
nextMonthFinished (Event) When the datepicker has finished the next month animation.
validDate (Event) When a user has manually typed a date and that date is valid.
invalidDate (Event) When a user has manually typed a date and that date is not valid.
